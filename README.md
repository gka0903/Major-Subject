<details> <summary>프로세스 동기화: 뮤텍스 & 세마포어</summary>
2.1 뮤텍스 (Mutex: Mutual Exclusion)

<details> <summary>▶ 뮤텍스 상세 내용 보기 (정의, 소유권, 특징)</summary>

공유 자원을 점유하기 위한 '열쇠(Key)' 개념입니다.

    정의: 오직 한 개의 프로세스 또는 스레드만이 공유 자원에 접근할 수 있도록 제어하는 상호 배제 기법입니다.

    소유권(Ownership): 가장 중요한 특징. 락(Lock)을 획득한 주체만이 그 락을 해제할 수 있습니다.

    상태: Locked와 Unlocked 두 가지 상태만 존재합니다.

    비유: 화장실이 하나뿐인 식당에서 화장실 열쇠를 들고 간 사람이 나올 때까지 아무도 못 들어가는 것과 같습니다.

    특징: 안전성이 높지만, 한 번에 한 명만 가능하므로 처리량이 제한될 수 있습니다.

</details>
2.2 세마포어 (Semaphore)

<details> <summary>▶ 세마포어 상세 내용 보기 (정의, 연산, 종류)</summary>

가용한 공유 자원의 개수를 나타내는 '신호기/계수기' 개념입니다.

    정의: 공유 자원에 접근할 수 있는 프로세스/스레드의 개수를 제한하는 방식입니다.

    종류:

        바이너리 세마포어: 0과 1의 값만 가짐 (뮤텍스와 유사하나 소유권이 없음).

        카운팅 세마포어: 0 이상의 정수 값을 가짐 (여러 개의 자원 관리).

    핵심 연산:

        P 연산 (Wait): 세마포어 값을 감소시킴. 값이 0이면 자원이 생길 때까지 대기.

        V 연산 (Signal): 세마포어 값을 증가시킴. 대기 중인 프로세스를 깨움.

    특징: 소유권 없음. 락을 걸지 않은 다른 프로세스도 Signal을 보내 자원을 해제하거나 대기자를 깨울 수 있습니다.

</details>
2.3 뮤텍스 vs 세마포어 핵심 비교

<details> <summary>▶ 비교 표 확인하기</summary>

비교 항목	뮤텍스 (Mutex)	세마포어 (Semaphore)
핵심 목적	상호 배제 (1:1)	자원 개수 관리 (1:N)
소유권	있음 (Lock을 잡은 주체만 해제)	없음 (누구나 Signal 가능)
허용 개수	1개	1개 이상 (설정 가능)
해제 주체	소유자가 반드시 해제해야 함	다른 프로세스가 해제 가능
범위	보통 프로세스 내 스레드 간 사용	시스템 전체(여러 프로세스) 사용 가능

</details>
2.4 시험 대비 심화 개념

<details> <summary>▶ 심화 개념 (우선순위 역전, 데드락, 사용 시나리오)</summary>

① 우선순위 역전 (Priority Inversion)

    현상: 낮은 우선순위 TL​이 뮤텍스를 잡고 있는데, 중간 우선순위 TM​이 CPU를 선점하여, 높은 우선순위 TH​가 무한정 대기하는 현상입니다.

    해결책: 우선순위 상속(Priority Inheritance). TL​의 우선순위를 일시적으로 TH​만큼 높여 빠르게 작업을 끝내고 락을 반납하게 합니다.

② 데드락 (Deadlock)과의 관계

    뮤텍스나 세마포어를 잘못 사용하여 **순환 대기(Circular Wait)**가 발생하면 시스템이 멈춥니다.

    시험 포인트: "세마포어 값을 잘못 조절했을 때(예: P 연산 중복 수행) 발생하는 문제"

③ 사용 시나리오

    뮤텍스: 전역 변수, 정적 변수 등 메모리 내 단일 자원을 보호할 때.

    세마포어: DB 커넥션 풀(개수 제한), 프린터 제한 등 여러 리소스를 관리할 때.

</details>
